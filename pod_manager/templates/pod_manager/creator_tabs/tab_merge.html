{% comment %} Merge Desk pane. THREE levels, and the nesting is load-bearing — see docs/boosted-navigation.md and the divergence C notes in planned_features_htmx_base_swap.txt. 1. The PANE carries the #boosted-region swap, so the merge body's POST forms (Merge Episodes / Split Episode) boost into a clean region swap and the out-of-region floating player survives. They used to be hx-boost="false" — born that way in 7c1ad09, the same commit that introduced this pane — precisely because there was nowhere safe for them to inherit from, so merging hard-navigated and killed playback. 2. #merge-desk-body is a PLAIN container. It must stay plain. It is the parent of the loaded body, so any htmx attribute on it is inherited by everything inside — and hx-disinherit on it would be worse: htmx's disinherit returns "unset" for EVERY attribute lookup that passes through the element, not just the ones it sets, so a disinheriting parent strips hx-boost off the forms too (native submit, hard refresh, audio dead) and hx-target off the nav (which then falls back to the element itself and renders the body onto the link that was clicked). 3. The LOADER does the one-shot GET into #merge-desk-body and is destroyed by its own swap (it replaces its parent's innerHTML), taking hx-disinherit="*" with it. That is why _lazy_pane can use the same trick safely: its loader targets the pane and disappears. A loader that targeted ITSELF would survive as the body's parent and poison it — which is exactly the bug this comment exists to prevent someone re-introducing. The merge desk's own in-tab nav (mode switch / search / the three paginators) therefore sets hx-target="#merge-desk-body" explicitly per element in _merge_desk_body.html. That is the right split of responsibility: swapping just this body is the special case and says so, while the forms get the sensible default (a region swap) by inheriting from the pane. {% endcomment %}
