{% comment %} THE episode audio upload. Shared by episode_detail.html and publish_episode.html — Josh: "my upload options are missing on the publish page, I can only paste links... we're already doing this on the episode edit suggestion page for the same exact episode audio." No migration and no new action: it posts the EXISTING manage_episode action='upload_audio', which mirrors the file to R2, attaches it to the Premium / Ad-Free slot and re-queues transcription. Params: action_url manage_episode URL for the episode (required) help_text small print under the field button_label submit label (default "Upload & Mirror") IT IS A FORM, unlike snippets/_image_upload.html — the audio upload is always standalone, so it owns its
rather than embedding in one. That is what lets the publish page host it at all: its Audio Files card lives INSIDE the big publish form, and a nested form is invalid HTML, so this sits after that form closes. It is BOOSTED, upload and all. This form used to be hx-boost="false" on the episode page so the browser could "show upload progress" — but a browser shows NO progress for a form POST (you get a spinning tab and a dead page), and the hard navigation killed playback, which is the one thing the boosted-navigation system exists to prevent. enctype stays for the no-JS submit; hx-encoding is what makes htmx send it as FormData over XHR, which both keeps the player alive and exposes the progress events _upload_progress.html reads.
