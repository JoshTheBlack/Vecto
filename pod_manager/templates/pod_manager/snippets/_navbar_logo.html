{% comment %} The navbar brand's logo: the network's logo if it has one, else the Vecto mark. Rendered in TWO places and they must not drift: - base.html, inside the persistent nav; - creator_tabs/tab_network.html, inside an hx-swap-oob wrapper, so uploading or removing a logo actually updates the nav. Why the OOB is needed at all: the nav lives OUTSIDE #boosted-region, so nothing re-renders it after a swap. Changing a nav-visible value server-side and expecting it to appear is a silent no-op — the nav would keep showing the old logo until a real page load. See "The nav persists" in docs/boosted-navigation.md (the avatar picker is the other live example). It renders the FALLBACK too, not just the img: removing a logo has to put the Vecto mark back, and an OOB fragment that only knew how to render an would leave the old logo sitting there on removal. Reads display_logo, which resolves upload > theme_config.logo_url > logo_url. Never read those fields directly here — an upload would be silently ignored.
