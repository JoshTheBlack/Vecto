{% comment %} Shared Quill rich-text editor. Used by publish_episode.html and episode_detail.html — the two pages had divergent hand-rolled setups; this is the merge, standardised on the publish page's pattern. Params (all optional): quill_content HTML string to seed the editor with. Serialized through json_script, NEVER rendered into the editor div. quill_class classes for the editor div (default: bg-dark text-light) quill_placeholder placeholder text quill_toolbar JSON array literal overriding the default toolbar Reading the value back out: the initialised Quill instance is exposed as a `_quill` expando on both the mount and the editor div. Resolve it from a scoped root (see below) and read `quill.root.innerHTML` — that half was always correct on both pages and is unchanged. WHY THE EDITOR DIV IS SERVER-RENDERED EMPTY (the publish pattern; episode_detail used to render {{ ep.clean_description|safe }} straight into it): - init is idempotent BY CONSTRUCTION — the container's server state is always empty, so a re-run can't nest Quill inside Quill; - no FOUC of raw HTML before Quill boots; - content stays DATA rather than DOM, so Quill's own chrome can never be parsed back as content and saved into the DB; - json_script escapes the content instead of trusting |safe. dangerouslyPasteHTML is not a hack — it is Quill's documented API for loading HTML, and runs the same clipboard matchers Quill uses when parsing innerHTML at init. The content is already server-sanitised by sanitize_user_html. NO id ON THE EDITOR DIV, DELIBERATELY — see docs/boosted-navigation.md. htmx's settle phase reverts an id'd element's attributes to the SERVER's version ~20ms after a swap (for any id present in both the old and new content). Quill stores its state in classes on this element (ql-container ql-snow), so an id here means a same-page swap leaves Quill's DOM built but its classes stripped — every Quill rule is scoped under .ql-snow, so the link tooltip stops being hidden and renders as a stray text box you can type into but never save. Select by data attribute. Pinned by HtmxSettleClassStrippingTests. {% endcomment %}
