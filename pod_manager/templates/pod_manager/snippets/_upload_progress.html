{% comment %} THE upload progress bar. One implementation, shared by every upload in the app — snippets/_image_upload.html (network logo + fallback, the 404 pool, both mix covers, the avatar) and snippets/_audio_upload.html (episode audio, on both the episode page and the publish page). Params: processing_label the >=100% state (default "Processing…") CONTRACT. Include this inside an element marked data-upload-scope, which must contain exactly one and must itself be inside the form that does the uploading. The scope is what lets two widgets in one form each track their own field. The form MUST carry: enctype="multipart/form-data" hx-encoding="multipart/form-data" and must NOT set hx-boost="false". hx-encoding is not optional decoration: it is what makes htmx send the form as FormData over XHR, which is BOTH what keeps the out-of-region audio player alive on submit AND the only reason there are progress events to read. A browser shows NO progress for a native form POST — the "native progress" this replaced never existed. See docs/boosted-navigation.md. {% endcomment %}
